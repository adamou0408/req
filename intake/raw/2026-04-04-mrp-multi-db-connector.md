# 💡 我有一個想法

**我是誰**：群聯電子 大數據部門，將與 MIS 部門合作

**我想要**：
我想要設計一個MRP 系統，會有原始tiptop的oracle資料庫，以及其他的資料來源postgresql，但我不是知道全部的資料來源，所以專案會持續有資料來源的對接，希望可以從網頁輸入db 的帳密，讓後可讀取資料表的清單，以及function的清單，這樣才可以持續迭代。

專案會用在群聯電子的的製造生產，市場部，測試部門，業務，PM，HW rd，FW，等等部門的交換資訊上面。

**為什麼**：
群聯電子多個部門之間需要交換資訊，目前有 Tiptop ERP 系統（Oracle 資料庫）作為主要資料來源，但還有其他 PostgreSQL 資料來源，且未來會持續增加新的資料來源。需要一個統一的 MRP 系統來整合這些分散的資料，讓各部門（製造生產、市場部、測試部門、業務、PM、HW RD、FW 等）能有效率地共享和使用資料。

**補充**：
- 主要資料來源：Tiptop ERP（Oracle）
- 其他資料來源：PostgreSQL（可能還有更多未知來源）
- 關鍵功能需求：網頁介面輸入 DB 帳密 → 自動讀取資料表清單 + function 清單
- 使用部門：製造生產、市場部、測試部門、業務、PM、HW RD、FW
- 設計原則：資料來源會持續增加，系統必須支援動態對接
- 專案範圍：MRP 是第一個要解決的問題，但整體是更廣義的「跨部門資料整合平台」
- 資料方向：目前單向讀取，後續會由 MIS 協助開發 Tiptop 寫回功能
- 協作模式：大數據部門主導 + MIS 部門協作（MIS 負責 Tiptop/Oracle 端）
