# 💡 兩個新需求

**我是誰**：大數據部門

---

## 需求一：同步監控設定可視化

**我想要**：
同步監控有看到 CDC 與 Batch，設定頁面要設定成前端可操作的設定介面，不然發生錯誤時不知道是跟哪個設定有關。

**為什麼**：
目前同步監控只能看到狀態，但 CDC 和 Batch 的設定（排程、頻率、連線參數等）沒有前端介面可以操作。如果同步出錯，使用者無法從前端確認是哪個設定導致的問題，必須去看後端設定檔，這對非工程師來說很不友善。

**補充**：
- 需要能在前端新增/修改/刪除同步設定
- CDC 和 Batch 要有不同的設定表單
- 出錯時能看到錯誤訊息和對應的設定

---

## 需求二：ETL 排程 + BI 報表 + 自訂 Dashboard

**我想要**：
既然有 DB 連線，希望可以有個排程做 ETL，讓其他有權限的使用者自己做排程整理資料，並且可以使用 BI 的功能做出固定的報表。可以 pipeline 做 chart 或 table，更進一步功能要做 chart 與 table 可以組合成 dashboard 並且固定下來。

**為什麼**：
目前系統只有預設的儀表板（PM、品質、同步），但各部門可能有自己想要追蹤的指標和資料組合。如果使用者可以自己定義 ETL 排程、拉出報表、組成自訂 dashboard，就不用每次有新的報表需求都要請大數據部門開發。

**補充**：
- ETL：使用者能在網頁上設定資料來源 → 轉換規則 → 目標表，設定排程自動執行
- BI 報表：使用者能選擇資料表，拖拉做出 chart（折線圖、柱狀圖、圓餅圖等）或 table
- Dashboard 組裝：多個 chart 和 table 可以組合成一個 dashboard 頁面
- 固定下來：dashboard 可以儲存、命名、分享給其他人
- 權限控制：依角色決定誰能建立/編輯/查看 dashboard
