## 需求摘要
群聯電子需要一個跨部門資料整合平台，第一階段解決 MRP（物料需求計畫）問題。平台需整合 Tiptop ERP（Oracle）、PostgreSQL 及未來持續增加的資料來源。核心功能是提供網頁介面讓管理者輸入資料庫連線資訊，自動探索資料表和 function 清單，使各部門（製造、市場、測試、業務、PM、HW RD、FW）能在統一平台上交換資訊。目前為單向讀取，後續視明確需求由 MIS 協助開發 Tiptop 寫回功能。

## 已確認的設計決策
- **MRP 運算邏輯**：自行開發 MRP II（Manufacturing Resource Planning），第一階段涵蓋經營規劃、銷售規劃、能力需求計劃（CRP）。**財務管理不在第一階段範圍**，後續視需求擴展
- **資料更新策略**：CDC 即時 + 批次混用（依資料類型決定）
- **身份驗證**：整合 Active Directory（AD），後續可擴展 SSO
- **Tiptop 寫回**：不在第一階段範圍，需有明確需求才會請 Tiptop 人員開發
- **資料來源**：目前已知 Oracle（Tiptop）+ PostgreSQL，其他待發現，架構需支援動態新增
- **Flash 採購資料**：來自 Tiptop 採購模組
- **主力組合變更**：需市場部主管核准後才能發布
- **衝突解決方針**：全面以「先有效果、快速展現價值」為優先，大數據探索優先、即時查詢優先、跨部門可見性優先、市場靈活切換優先

## 使用者故事

### 大數據工程師（Big Data Engineer）
- **作為**大數據工程師，**我想要**在網頁介面上輸入資料庫的連線資訊（類型、主機、帳號、密碼），**以便**快速對接新的資料來源而不需要改程式碼
- **驗收條件**：
  - [ ] 支援輸入 Oracle 和 PostgreSQL 的連線資訊
  - [ ] 連線資訊以加密方式儲存
  - [ ] 可測試連線是否成功
  - [ ] 支援未來擴充新的資料庫類型（插件式架構）

- **作為**大數據工程師，**我想要**連線成功後自動列出該資料庫的所有資料表和 function/stored procedure 清單，**以便**快速了解資料結構並規劃整合方案
- **驗收條件**：
  - [ ] 顯示所有資料表名稱、欄位名稱與資料類型
  - [ ] 顯示所有 function / stored procedure 清單及參數
  - [ ] 可以預覽每個資料表的前 N 筆資料
  - [ ] 支援搜尋和篩選資料表/欄位

- **作為**大數據工程師，**我想要**定義跨資料來源的資料對應關係（mapping），**以便**建立統一的資料檢視供各部門使用
- **驗收條件**：
  - [ ] 可設定不同資料來源的欄位對應
  - [ ] 支援資料轉換規則（型別轉換、值映射）
  - [ ] 對應關係可版本控制

- **作為**大數據工程師，**我想要**設定資料同步策略（即時 CDC 或定時批次），**以便**依資料類型選擇最適合的更新方式
- **驗收條件**：
  - [ ] 可對每個資料來源/資料表個別設定同步模式（CDC / 排程批次）
  - [ ] CDC 支援 Oracle LogMiner 或 PostgreSQL logical replication
  - [ ] 批次同步支援設定排程（每 N 分鐘 / 每小時 / 每日）
  - [ ] 同步狀態可監控（最後同步時間、是否有延遲）

### MIS 系統管理員（MIS Administrator）
- **作為** MIS 管理員，**我想要**控制哪些資料表和欄位可以被外部存取，**以便**保護 Tiptop/Oracle 的資料安全和系統效能
- **驗收條件**：
  - [ ] 可設定白名單：只有被允許的資料表/欄位才能被讀取
  - [ ] 可設定讀取頻率限制，避免拖垮 ERP
  - [ ] 所有存取行為都有完整的 audit log
  - [ ] 可隨時停用某個連線

- **作為** MIS 管理員，**我想要**看到所有外部系統對 Tiptop 資料庫的存取紀錄，**以便**監控系統負載和排查問題
- **驗收條件**：
  - [ ] 顯示每次查詢的時間、來源、查詢內容、回應時間
  - [ ] 異常查詢（過慢、過頻）自動告警
  - [ ] 可匯出存取紀錄報表

### 製造生產人員（Manufacturing Staff）
- **作為**製造生產人員，**我想要**在統一平台上看到 MRP 運算結果（物料需求、缺料狀況、建議採購/生產排程），**以便**快速做出生產決策
- **驗收條件**：
  - [ ] 顯示物料需求清單（依產品/時間維度）
  - [ ] 標示缺料預警（現有庫存 vs 需求量）
  - [ ] 支援 CDC 即時更新關鍵資料（庫存、在途）和批次更新非即時資料（歷史報表）

- **作為**製造生產人員，**我想要**看到跨部門的關聯資訊（訂單、BOM、庫存、市場部的主力組合決策），**以便**做出更全面的排程決策
- **驗收條件**：
  - [ ] 從生產排程可以連結到對應訂單資訊
  - [ ] 從排程可以展開 BOM 查看物料組成
  - [ ] 可查看市場部當前定義的主力生產組合
  - [ ] 資料來自不同資料來源但呈現方式一致

### 市場部人員（Market Planning）
- **作為**市場部人員，**我想要**查看 Controller 與 NAND Flash 的搭配組合銷售數據和庫存狀況，**以便**決定主力生產組合
- **驗收條件**：
  - [ ] 可查看各 Controller + Flash 組合的銷售量和趨勢
  - [ ] 可查看各組合的當前庫存和在途數量
  - [ ] 可查看 Flash 供應商的交期和價格趨勢
  - [ ] 支援匯出報表（Excel/PDF）

- **作為**市場部人員，**我想要**在平台上發布和管理主力生產組合決策，**以便**讓業務、製造等部門即時取得最新方向
- **驗收條件**：
  - [ ] 可建立/修改/發布主力組合清單（Controller 型號 + Flash 型號 + 目標比例）
  - [ ] 發布後自動通知相關部門（業務、製造）
  - [ ] 保留歷史決策紀錄（誰、何時、改了什麼）
  - [ ] 製造和業務的頁面可即時看到最新主力組合

- **作為**市場部人員，**我想要**管理 NAND Flash 的採購資訊，**以便**追蹤採購進度和成本
- **驗收條件**：
  - [ ] 可查看各 Flash 型號的採購訂單、到料狀況
  - [ ] 可查看 Flash 價格歷史趨勢
  - [ ] 庫存低於安全水位時自動預警
  - [ ] 可整合 Tiptop 採購模組的資料

### 業務人員（Sales）
- **作為**業務人員，**我想要**即時查詢庫存和生產排程狀態，並看到市場部最新的主力組合方向，**以便**快速且準確地回覆客戶交期，並優先推薦主力產品
- **驗收條件**：
  - [ ] 可用產品型號搜尋庫存數量
  - [ ] 可查看該產品的生產排程和預計完成日
  - [ ] 首頁顯示市場部當前的主力 Controller+Flash 組合
  - [ ] 介面簡潔，不需要懂資料庫操作
  - [ ] 支援手機瀏覽

- **作為**業務人員，**我想要**看到各產品組合的銷售數據，**以便**配合市場部的方向進行銷售
- **驗收條件**：
  - [ ] 可依產品組合、區域、時間維度查看銷售趨勢
  - [ ] 主力組合在報表中特別標示
  - [ ] 支援匯出報表（Excel/PDF）

### 專案經理（PM）
- **作為** PM，**我想要**在一個儀表板上看到產品從設計到量產各階段的進度和關鍵指標，**以便**及時發現瓶頸並協調資源
- **驗收條件**：
  - [ ] 儀表板顯示專案進度、測試良率、生產狀態
  - [ ] 支援多專案同時追蹤
  - [ ] 異常指標自動標紅/告警
  - [ ] 資料來自多個系統但統一呈現

### 硬體 / 韌體研發工程師（HW RD / FW）
- **作為** HW/FW 工程師，**我想要**查詢 BOM 資訊和工程變更歷史，**以便**確認當前版本的物料是否正確
- **驗收條件**：
  - [ ] 可用料號搜尋 BOM
  - [ ] 顯示 BOM 變更歷史（ECN 紀錄）
  - [ ] 可與測試結果交叉比對（哪個 BOM 版本對應哪批測試）

- **作為** FW 工程師，**我想要**查看不同韌體版本對應的測試結果，**以便**分析版本間的差異和問題
- **驗收條件**：
  - [ ] 可選擇韌體版本查看對應測試數據
  - [ ] 支援版本間測試結果比較
  - [ ] 顯示良率趨勢圖

### 測試部門（QA / Test Engineer）

- **驗收條件**：
  - [ ] 支援從多個測試資料來源匯整數據
  - [ ] 可依批次、產品、時間維度分析良率
  - [ ] 不良品可追溯到 BOM 版本和生產批次
  - [ ] 支援匯出分析報告

## 非功能需求
- 效能：資料查詢回應時間 < 3 秒（常用查詢）；不得造成 Tiptop ERP 效能下降 > 5%
- 安全性：DB 帳密加密儲存；整合 AD 做身份驗證；角色權限管控（RBAC）；所有存取行為記錄 audit log
- 相容性：支援 Oracle、PostgreSQL；架構可擴展支援 MySQL、SQL Server、API 等
- 可用性：網頁介面支援 Chrome/Edge；支援手機瀏覽（基本查詢功能）
- 資料同步：支援 CDC 即時同步 + 定時批次同步混用
