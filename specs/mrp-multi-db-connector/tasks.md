## Phase 1：基礎平台

### Task 1.1：專案腳手架與開發環境
- **對應 User Story**：全部（基礎建設）
- **描述**：初始化前後端專案結構、Docker Compose 開發環境、CI/CD pipeline
- **驗收條件**：
  - [ ] FastAPI 後端可啟動並回應 `/health`
  - [ ] React 前端可啟動並顯示登入頁面
  - [ ] Docker Compose 一鍵啟動所有服務（API + DB + Redis）
  - [ ] CI pipeline 可執行 lint + test
- **狀態**：`todo`

### Task 1.2：AD 身份驗證
- **對應 User Story**：MIS 管理員 — 存取控管
- **描述**：整合 Active Directory (LDAP) 做使用者驗證，JWT token 做 API 授權
- **驗收條件**：
  - [ ] 使用者可用 AD 帳號密碼登入
  - [ ] 登入成功取得 JWT token
  - [ ] Token 含使用者角色資訊
  - [ ] 支援 token refresh
  - [ ] 登入失敗有明確錯誤訊息
- **狀態**：`todo`

### Task 1.3：RBAC 權限模型
- **對應 User Story**：MIS 管理員 — 存取控管；各部門 — 角色權限
- **描述**：實作角色權限管控，含 8 個角色模板和敏感欄位黑名單
- **驗收條件**：
  - [ ] 8 個角色模板已建立（big_data, mis, manufacturing, market, sales, pm, hw_fw_rd, qa）
  - [ ] 權限矩陣按 spec 中的建議矩陣實作
  - [ ] 敏感欄位黑名單可管理（薪資、成本、個資）
  - [ ] API 層自動依角色過濾回傳資料
- **狀態**：`todo`

### Task 1.4：DB 連線管理（CRUD）
- **對應 User Story**：大數據工程師 — 網頁輸入 DB 連線資訊
- **描述**：網頁介面管理資料庫連線（新增、修改、測試、停用）
- **驗收條件**：
  - [ ] 可輸入 Oracle 連線資訊（host, port, SID/service, username, password）
  - [ ] 可輸入 PostgreSQL 連線資訊
  - [ ] 密碼以 AES-256-GCM 加密儲存
  - [ ] 可測試連線是否成功（顯示成功/失敗 + 錯誤原因）
  - [ ] 可停用/啟用連線
  - [ ] 插件式架構：新增 DB 類型只需實作 `DBConnector` 介面
- **狀態**：`todo`

### Task 1.5：Schema Explorer（資料表/Function 探索）
- **對應 User Story**：大數據工程師 — 自動列出資料表和 function 清單
- **描述**：連線成功後自動探索資料庫結構，顯示表/欄位/function
- **驗收條件**：
  - [ ] 顯示所有資料表名稱（含 schema/owner 分類）
  - [ ] 每個表顯示欄位名稱、資料類型、是否 nullable、PK/FK
  - [ ] 顯示 function / stored procedure 清單及參數簽名
  - [ ] 可預覽每個表的前 N 筆資料（預設 10 筆）
  - [ ] 支援搜尋和篩選（表名、欄位名）
  - [ ] Oracle 和 PostgreSQL 都能正常運作
- **狀態**：`todo`

### Task 1.6：Audit Log 系統
- **對應 User Story**：MIS 管理員 — 存取紀錄
- **描述**：記錄所有存取行為，提供查詢和匯出功能
- **驗收條件**：
  - [ ] 每次 API 呼叫記錄：使用者、時間、動作、目標、回應時間
  - [ ] 查詢 DB 時記錄完整 query 內容
  - [ ] 可依時間、使用者、動作類型篩選
  - [ ] 異常偵測：過慢（> 3s）或過頻（> 60/min）自動標記
  - [ ] 可匯出為 CSV/Excel
- **狀態**：`todo`

## Phase 2：資料同步 + 市場業務

### Task 2.1：批次同步引擎
- **對應 User Story**：大數據工程師 — 設定資料同步策略（批次部分）
- **描述**：用 Celery Beat 實作定時資料同步
- **驗收條件**：
  - [ ] 可為每個表設定同步排程（每 N 分鐘 / 每小時 / 每日）
  - [ ] 同步執行時顯示進度
  - [ ] 記錄最後同步時間和狀態
  - [ ] 同步失敗自動重試（最多 3 次）+ 告警
  - [ ] 可手動觸發同步
- **狀態**：`todo`

### Task 2.2：CDC 即時同步引擎
- **對應 User Story**：大數據工程師 — 設定資料同步策略（CDC 部分）
- **描述**：用 Debezium 實作 Oracle 和 PostgreSQL 的即時變更捕捉
- **驗收條件**：
  - [ ] Oracle CDC 可捕捉 INSERT/UPDATE/DELETE 變更
  - [ ] PostgreSQL CDC 可捕捉即時變更
  - [ ] 變更事件即時寫入 platform DB
  - [ ] 同步延遲可監控（目標 < 30 秒）
  - [ ] CDC 連線中斷自動重連
- **狀態**：`todo`

### Task 2.3：欄位對應（Field Mapping）
- **對應 User Story**：大數據工程師 — 定義跨資料來源的資料對應關係
- **描述**：網頁介面定義不同資料來源的欄位對應和轉換規則
- **驗收條件**：
  - [ ] 可視覺化拖拉設定欄位對應（source → target）
  - [ ] 支援轉換規則：型別轉換、值映射、簡單運算
  - [ ] 對應關係可儲存和載入
  - [ ] 支援版本控制（可查看歷史版本、回滾）
- **狀態**：`todo`

### Task 2.4：主力組合管理
- **對應 User Story**：市場部 — 發布和管理主力生產組合
- **描述**：市場部建立、審核、發布 Controller+Flash 主力組合
- **驗收條件**：
  - [ ] 可建立主力組合（Controller 型號 + Flash 型號 + 目標比例）
  - [ ] 送出後需市場部主管核准
  - [ ] 核准後可發布，自動通知業務和製造（WebSocket 推送）
  - [ ] 保留完整歷史紀錄（誰、何時、改了什麼）
  - [ ] 業務和製造頁面即時顯示當前主力組合
- **狀態**：`todo`

### Task 2.5：庫存即時查詢
- **對應 User Story**：業務 — 即時查詢庫存和排程；市場部 — 庫存狀況
- **描述**：從 Tiptop 同步庫存資料，提供即時查詢介面
- **驗收條件**：
  - [ ] 可用產品型號/料號搜尋庫存數量
  - [ ] 顯示各倉庫庫存明細
  - [ ] 顯示在途數量
  - [ ] 回應時間 < 3 秒
  - [ ] 支援手機瀏覽
- **狀態**：`todo`

### Task 2.6：Flash 採購追蹤
- **對應 User Story**：市場部 — 管理 NAND Flash 採購資訊
- **描述**：從 Tiptop 採購模組同步 Flash 採購資料
- **驗收條件**：
  - [ ] 顯示各 Flash 型號的採購訂單和到料狀況
  - [ ] 顯示價格歷史趨勢圖
  - [ ] 庫存低於安全水位自動預警
  - [ ] 資料來自 Tiptop 採購模組
- **狀態**：`todo`

### Task 2.7：銷售報表
- **對應 User Story**：業務 — 銷售數據；市場部 — 組合銷售分析
- **描述**：銷售趨勢分析和報表匯出
- **驗收條件**：
  - [ ] 可依產品組合、區域、時間維度查看趨勢
  - [ ] 主力組合在報表中特別標示
  - [ ] 支援匯出 Excel 和 PDF
  - [ ] Controller+Flash 組合維度的銷售分析
- **狀態**：`todo`

## Phase 3：MRP 運算

### Task 3.1：BOM 展開與管理
- **對應 User Story**：HW/FW RD — BOM 查詢和 ECN 歷史
- **描述**：從 Tiptop 同步 BOM 資料，支援多階 BOM 展開
- **驗收條件**：
  - [ ] 可用料號搜尋 BOM
  - [ ] 支援多階 BOM 展開（樹狀顯示）
  - [ ] 顯示 ECN 變更歷史
  - [ ] BOM 版本可與測試結果交叉比對
- **狀態**：`todo`

### Task 3.2：MRP 淨需求運算
- **對應 User Story**：製造生產 — MRP 運算結果
- **描述**：實作 MRP I 核心運算：毛需求 → 淨需求 → 建議訂單
- **驗收條件**：
  - [ ] 從銷售訂單/預測計算毛需求
  - [ ] 扣除庫存和在途計算淨需求
  - [ ] 依 BOM 展開計算各階物料需求
  - [ ] 依提前期（lead time）計算建議採購/生產日期
  - [ ] 顯示物料需求清單（依產品/時間維度）
  - [ ] 缺料預警：現有供給 < 需求
- **狀態**：`todo`

### Task 3.3：主生產排程（MPS）
- **對應 User Story**：製造生產 — 生產排程；PM — 進度追蹤
- **描述**：依 MRP 結果和市場部主力組合生成主生產排程
- **驗收條件**：
  - [ ] 依主力組合比例和 MRP 需求生成排程建議
  - [ ] 排程可連結到對應訂單
  - [ ] 可查看市場部當前主力組合
  - [ ] 排程變動有歷史紀錄
- **狀態**：`todo`

### Task 3.4：CRP 能力需求計劃
- **對應 User Story**：製造生產 — 排程決策；PM — 瓶頸發現
- **描述**：分析產能是否滿足 MPS，識別瓶頸
- **驗收條件**：
  - [ ] 計算各工作中心/產線的負荷
  - [ ] 負荷超過產能時自動告警
  - [ ] 產能瓶頸視覺化圖表
  - [ ] 提供產能調整建議
- **狀態**：`todo`
