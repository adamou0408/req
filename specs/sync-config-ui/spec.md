## 需求摘要
目前同步監控頁面只能看到 CDC/Batch 狀態，但設定（排程、頻率、連線參數）沒有前端介面可以操作。需要把同步設定做成前端可視化的管理介面，讓使用者能新增、修改、刪除同步設定，並且在出錯時能直接看到是哪個設定導致問題。

## 已確認的設計決策
- 後端 API 已存在：`/api/sync/configs` CRUD + `/api/sync/status`
- 只需新增前端頁面，**後端不需要大改**

## 使用者故事

### 大數據工程師（Big Data Engineer）
- **作為**大數據工程師，**我想要**在前端頁面新增和管理同步設定，**以便**不用改設定檔就能對接新資料
- **驗收條件**：
  - [ ] 可新增同步設定：選擇資料來源 → 選擇表 → 選擇模式（CDC/Batch）→ 設定參數
  - [ ] CDC 模式：顯示相關設定（自動偵測變更、追蹤欄位等）
  - [ ] Batch 模式：可設定 cron 排程（提供視覺化排程選擇器，不用手寫 cron）
  - [ ] 可修改和停用現有設定
  - [ ] 設定變更有歷史紀錄

- **作為**大數據工程師，**我想要**在同步出錯時看到錯誤詳情和對應的設定，**以便**快速定位問題
- **驗收條件**：
  - [ ] 錯誤狀態的同步設定以紅色標示
  - [ ] 點擊可展開看到：錯誤訊息、發生時間、相關設定參數
  - [ ] 可一鍵重試失敗的同步
  - [ ] 錯誤歷史紀錄可查詢

### MIS 系統管理員（MIS Administrator）
- **作為** MIS 管理員，**我想要**在前端控制 Tiptop 相關的同步設定，**以便**在 ERP 維護時快速停用同步
- **驗收條件**：
  - [ ] 可按資料來源篩選同步設定
  - [ ] 可一鍵停用/啟用某個資料來源的所有同步
  - [ ] 顯示每個同步對 ERP 的查詢頻率
