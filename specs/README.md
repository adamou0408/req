# 📋 結構化需求規格（Specs）

## 這是什麼？

這裡存放的是 AI 從原始需求轉譯出來的結構化規格。每個功能有一個獨立的資料夾，裡面包含規格、技術方案和任務清單。

## 規格的生命週期

每份規格會經歷以下狀態：

```
draft → in-review → approved → in-progress → done
```

| 狀態 | 意義 |
|------|------|
| `draft` | AI 剛轉譯完成，等待審核 |
| `in-review` | 正在審核中 |
| `approved` | 人類審核通過，可以開始開發 |
| `in-progress` | AI 正在實作中 |
| `done` | 開發完成，測試通過 |

## 每個功能資料夾包含

| 檔案 | 用途 |
|------|------|
| `spec.md` | 需求規格（User Story、驗收條件、衝突標記） |
| `plan.md` | 技術實作方案（審核通過後才會產生） |
| `tasks.md` | 任務清單（從 plan 拆解出來的） |

## 模板

`_template/` 資料夾中有各檔案的模板，AI 會自動使用這些模板來生成新的規格。
