# 業務人員（End User）

## 基本描述
- **誰是這個角色**：負責對外溝通的業務、行銷、客服人員
- **主要職責**：把產品資訊、活動連結送到客戶手上
- **技術熟悉度**：不懂（只會用完成品）
- **使用頻率**：每天多次

## 核心需求
- 最在乎：建立短網址要快、網址要能看懂、不要被系統卡住
- 最怕：網址長得像詐騙讓客戶不敢點、有額度限制打斷工作節奏
- 典型工作場景：活動促銷一次產生 10~30 個連結、LINE / Email / 社群群發、長期活動連結重用

## 與其他角色的關係
- 會和哪些角色互動：管理員（配額問題）、客服（使用者回報點不到）
- 常見的利益衝突點：想要無限制建立 vs 管理員想避免濫用；想自己取名 vs 管理員想避免衝撞品牌詞
