# 測試部門人員（QA / Test Engineer）

## 基本描述
- **誰是這個角色**：品質保證和測試工程師，負責產品驗證和良率管控
- **主要職責**：產品測試、良率分析、不良品追蹤、品質報告
- **技術熟悉度**：略懂（熟悉測試工具，略懂資料分析，不熟資料庫）
- **使用頻率**：每天

## 核心需求
- 最在乎：測試數據完整性、良率趨勢、不良原因分析、測試結果能快速關聯到對應批次和版本
- 最怕：測試數據散落各處無法彙整、良率異常找不到原因、無法追溯到哪個版本的 BOM 或 FW
- 典型工作場景：跑完一批測試 → 上傳結果 → 分析良率 → 異常回饋給 RD → 出品質報告

## 與其他角色的關係
- 會和哪些角色互動：HW RD / FW（回饋測試問題）、製造（生產批次資訊）、PM（品質進度）
- 常見的利益衝突點：要充分測試 vs PM 要趕交期；要退回重做 vs 製造要出貨；要詳細資料 vs 數據太多影響系統效能
