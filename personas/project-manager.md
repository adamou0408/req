# 專案經理（PM - Project Manager）

## 基本描述
- **誰是這個角色**：產品專案經理，負責產品從設計到量產的全流程管理
- **主要職責**：產品進度追蹤、跨部門協調、資源調度、交期管控、風險管理
- **技術熟悉度**：略懂（看得懂報表，了解流程，但不操作資料庫）
- **使用頻率**：每天

## 核心需求
- 最在乎：專案整體進度、各部門是否按時完成、瓶頸在哪、風險預警
- 最怕：資訊散落在不同系統看不到全貌、部門之間資訊不同步、問題被隱藏到最後一刻才爆發
- 典型工作場景：開晨會看各專案狀態 → 追蹤瓶頸 → 協調資源 → 更新進度報告

## 與其他角色的關係
- 會和哪些角色互動：HW RD（設計進度）、FW（韌體進度）、測試（驗證進度）、製造（量產進度）、業務（客戶需求）
- 常見的利益衝突點：要看到所有細節 vs 各部門想控制資訊揭露；要加速進度 vs RD 要時間做好；要同時追多個專案 vs 資源有限
