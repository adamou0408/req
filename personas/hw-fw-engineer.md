# 硬體 / 韌體研發工程師（HW RD / FW Engineer）

## 基本描述
- **誰是這個角色**：硬體設計或韌體開發工程師，負責產品的技術設計與開發
- **主要職責**：IC 設計 / 韌體開發、BOM 管理、工程變更（ECN）、技術驗證
- **技術熟悉度**：熟練（技術能力強，但不一定熟悉 ERP 和資料庫）
- **使用頻率**：每週

## 核心需求
- 最在乎：BOM 正確性、工程變更是否被正確執行、測試數據、版本管控
- 最怕：BOM 版本混亂導致生產用錯料、工程變更沒通知到相關部門、找不到歷史測試數據
- 典型工作場景：更新 BOM → 發起工程變更 → 確認測試結果 → 簽核放行量產

## 與其他角色的關係
- 會和哪些角色互動：PM（進度報告）、測試部門（驗證結果）、製造（量產可行性）、MIS（系統資料）
- 常見的利益衝突點：要頻繁變更設計 vs 製造要穩定；要完整測試 vs PM 要趕進度；要詳細的技術資料 vs 業務只要簡單規格
