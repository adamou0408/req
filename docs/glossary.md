# 術語對照表

這份文件幫助非技術人員和技術人員使用相同的語言溝通。

## 一般用語 ↔ 技術用語

| 你可能會說... | 在這個專案裡叫做... | 說明 |
|--------------|-------------------|------|
| 「我想要...」 | 需求（Requirement） | 你對系統的期望或想法 |
| 「寫下我要什麼」 | 需求規格（Spec） | AI 把你的話整理成結構化的文件 |
| 「站在某人的角度」 | 使用者角色（Persona） | 系統中不同類型的使用者 |
| 「如果是我，我會想...」 | 使用者故事（User Story） | 從某個角色的角度描述需求 |
| 「怎麼算做好了」 | 驗收條件（Acceptance Criteria） | 功能完成的具體標準 |
| 「A 要這個但 B 要那個」 | 需求衝突（Conflict） | 不同角色之間的需求矛盾 |
| 「我看一下對不對」 | 審核（Review） | 人類確認 AI 整理的內容是否正確 |
| 「開始做了」 | 實作（Implementation） | AI 根據規格撰寫程式碼 |
| 「測試一下」 | 自動化測試（Automated Test） | 電腦自動檢查程式是否正確 |
| 「我改主意了」 | 需求變更（Iteration） | 修改之前提過的需求 |

## 狀態說明

| 狀態名稱 | 白話意思 |
|----------|---------|
| `draft` | 草稿 — AI 剛整理好，還沒人看過 |
| `in-review` | 審核中 — 有人正在看，確認是否正確 |
| `approved` | 通過 — 確認沒問題，可以開始做了 |
| `in-progress` | 進行中 — AI 正在寫程式 |
| `done` | 完成 — 做好了，測試也通過了 |
| `detected` | 發現衝突 — AI 找到了需求矛盾 |
| `under-discussion` | 討論中 — 大家正在討論怎麼解決衝突 |
| `resolved` | 已解決 — 衝突已經有結論了 |

## 檔案和資料夾

| 資料夾名稱 | 白話意思 |
|-----------|---------|
| `intake/` | 收需求的地方 |
| `intake/raw/` | 你寫的原始內容放這裡 |
| `personas/` | 使用者角色的介紹 |
| `specs/` | AI 整理好的需求規格 |
| `conflicts/` | 需求衝突的紀錄 |
| `reviews/` | 審核紀錄 |
| `src/` | 程式碼（AI 寫的） |
| `tests/` | 測試（AI 寫的，用來確認程式沒問題） |
| `docs/` | 專案文件 |
