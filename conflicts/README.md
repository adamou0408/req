# ⚠️ 需求衝突紀錄（Conflicts）

## 這是什麼？

當不同角色的需求互相矛盾時，AI 會自動偵測並在這裡建立紀錄。

## 衝突怎麼被偵測？

AI 在轉譯需求（`/translate`）時，會自動比對不同角色的 User Story，找出以下類型的衝突：

| 衝突類型 | 說明 | 範例 |
|----------|------|------|
| 功能衝突 | A 要 X，B 要 非-X | 用戶要公開資料，管理員要限制存取 |
| 優先級衝突 | 兩方都需要，但資源有限 | A 功能和 B 功能都要先做 |
| 權限衝突 | 一方要開放，一方要限制 | 開放 API vs 封閉系統 |
| UX 衝突 | 簡化操作 vs 豐富功能 | 一鍵完成 vs 提供更多選項 |

## 衝突怎麼解決？

1. AI 偵測到衝突後會標記為 `detected`
2. AI 會提供分析和建議方案（但不做決定）
3. **必須由人類來裁決**（這是流程中的關鍵人工環節）
4. 裁決後標記為 `resolved`，記錄決策理由

## 衝突的狀態

| 狀態 | 意義 |
|------|------|
| `detected` | AI 偵測到，等待討論 |
| `under-discussion` | 正在討論中 |
| `resolved` | 已由人類裁決 |
