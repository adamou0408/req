# ✅ 審核紀錄（Reviews）

## 這是什麼？

每份需求規格在正式進入開發之前，都必須經過人類審核。這個資料夾存放所有的審核紀錄。

## 審核流程

1. AI 將需求轉譯為結構化規格後，規格狀態為 `draft`
2. 執行 `/review` 命令，AI 會生成審核清單
3. 人類審核者逐項確認清單
4. **通過**：規格狀態更新為 `approved`，可以進入開發
5. **退回**：規格狀態回到 `draft`，附上修改建議

## 審核重點

- 所有 User Story 是否完整且合理
- 驗收條件是否具體且可測試
- 所有衝突是否已解決
- 是否有未回答的開放問題
- 與現有功能是否有重疊

## 為什麼需要人類審核？

AI 很擅長整理和分析，但最終的業務判斷必須由人來做。審核是確保 AI 正確理解需求的最重要環節。
